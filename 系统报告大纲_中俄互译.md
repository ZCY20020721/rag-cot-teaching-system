## 第一章：引言 / Глава 1: Введение

### 1.1 项目背景 / Предпосылки проекта
- 大语言模型（LLM）在教育领域的应用现状 / Современное состояние применения LLM в образовании

- RAG（检索增强生成）与 CoT（思维链）技术的互补性 / Комплементарность технологий RAG и CoT

- 构建基于教材知识的智能出题与批改系统 / Построение интеллектуальной системы генерации заданий и оценивания на основе учебных материалов
- 实现教师-学生双角色的完整教学闭环 / Реализация полного замкнутого цикла обучения с ролями преподавателя и студента
- 提供可解释的、分步骤的 AI 批改反馈 / Обеспечение объяснимой, пошаговой обратной связи от AI

### 2.2 核心模块详细分析 / Детальный анализ основных модулей

#### 2.2.1 RAG 检索引擎 (rag_engine.py) / Движок поиска RAG
**类结构 / Структура класса:**
- `RAGEngine.__init__(persist_dir)` — 初始化嵌入模型 (all-MiniLM-L6-v2, 384维, CPU), 文本分割器 (chunk_size=500, overlap=50), 连接 ChromaDB
- `RAGEngine.load_pdf(pdf_path) → int` — PDF 加载 + 文本切分 + 向量化 + 持久化, 返回 chunk 数量
- `RAGEngine.search(query, k=5) → List[str]` — 语义相似度检索, 返回 top-k 文本片段
- `RAGEngine.search_with_sources(query, k=5) → List[dict]` — 检索并标注来源
- `RAGEngine.get_chunks_count() → int` — 知识库规模查询

**关键技术细节 / Ключевые технические детали:**
- 文本分割器配置：chunk_size=500, chunk_overlap=50, separators=["\n\n", "\n", "。", ".", " ", ""]
- 嵌入模型：sentence-transformers/all-MiniLM-L6-v2, 本地 CPU 运行
- 向量存储：ChromaDB 持久化至 ./chroma_db 目录
- 增量加载：支持多次 load_pdf 追加到同一向量库

#### 2.2.2 CoT 批改引擎 (cot_grader.py) / Движок оценивания CoT
**类结构 / Структура класса:**
- `CoTGrader.__init__()` — 初始化 DeepSeek Chat API (temperature=0.1, max_tokens=4096), Pydantic 解析器
- `CoTGrader.generate_question(rag_context) → Optional[dict]` — 基于教材生成考题, 结构化 JSON 输出
- `CoTGrader.grade_answer(rag_context, standard_answer_points, student_answer) → Optional[dict]` — CoT 三步批改

**Pydantic 数据模型 / Модели данных Pydantic:**
- `AnswerPoint`: point, tag, max_score
- `QuestionOutput`: question, standard_answer_points, total_max_score
- `StepScore`: point_index, student_content, score, comment
- `GradingOutput`: step_scores, logic_score, logic_comment, total_score, feedback, weak_tags

**JSON 解析容错机制 / Механизм отказоустойчивости парсинга JSON:**
- 主解析：提取 LLM 输出中的 JSON（处理 ```json 代码块包裹）
- 失败重试：追加格式提醒后重新请求
- 最终降级：返回 None, 前端提示用户重试

#### 2.2.3 Prompt 工程设计 (prompts.py) / Проектирование промптов
**三个核心 Prompt 模板 / Три основных шаблона промптов:**
1. `QUESTION_GENERATION_PROMPT` — 出题模板
   - 角色设定：资深教育专家
   - 输入变量：{rag_context}（教材内容）
   - 约束：紧贴教材、2-4个得分点、知识点打标
   - 输出格式：JSON {question, standard_answer_points, total_max_score}

2. `COT_GRADING_PROMPT` — CoT 批改模板
   - 三步思维链：要点匹配 → 逻辑评估 → 综合反馈
   - 苏格拉底式反馈原则：不直接透露答案，用引导式提问
   - 输出格式：JSON {step_scores, logic_score, total_score, feedback, weak_tags}

3. `WEAKNESS_ANALYSIS_PROMPT` — 薄弱点分析模板
   - 输入：学生错题记录
   - 输出：薄弱领域 + 关联分析 + 学习建议

#### 2.2.4 数据库层 (db.py) / Слой базы данных
**6 张数据表 / 6 таблиц данных:**

| 表名 / Таблица | 用途 / Назначение | 关键字段 / Ключевые поля |
|---------------|-------------------|------------------------|
| users | 用户系统 | id, username, password_hash(SHA256), role(teacher/student) |
| exercises | 教师发布的习题 | id, teacher_id(FK), question, standard_answer_points(JSON), total_max_score |
| exam_records | 学生答题记录 | id, student_id(FK), exercise_id(FK), answer, step_scores(JSON), weak_tags(JSON) |
| knowledge_tags | 知识点错误统计 | tag_name(UNIQUE), error_count |
| messages | 聊天消息 | id, sender_id(FK), receiver_id(FK), content, file_path, file_type |

**核心函数 / Основные функции:**
- 用户模块：`register_user()`, `login_user()`, `get_all_students()`, `get_all_teachers()`
- 习题模块：`save_exercise()`, `get_all_exercises()`, `get_exercise_by_id()`
- 答题模块：`save_exam_record()`, `get_student_records()`, `get_all_exam_records()`

## 第三章：实现层分析 / Глава 3: Анализ уровня реализации

### 3.1 RAG 检索增强生成流水线 / Конвейер RAG

#### 3.1.1 文档处理流程 / Процесс обработки документов
```
PDF文件 → PyPDFLoader → Document对象列表
  → RecursiveCharacterTextSplitter (500字/块, 50字重叠)
  → HuggingFaceEmbeddings (384维向量)
  → ChromaDB (持久化 + 索引)
  → similarity_search (余弦相似度)
  → Top-K 相关文本片段 → 拼接入 Prompt
```
Описание каждого этапа: ...

#### 3.1.2 防幻觉策略 / Стратегия предотвращения галлюцинаций
- 教材锚定：所有题目和批改均基于检索到的教材原文
- 最小检索量：k=5 保证足够上下文
- Prompt 约束：明确要求 "紧贴教材内容，不要超出教材范围"

### 3.2 CoT 思维链批改机制 / Механизм оценивания CoT

#### 3.2.1 三步推理链 / Трехшаговая цепочка рассуждений
```
步骤1: 要点匹配 (Per-Point Matching)
  学生答案 → 逐条对比标准答案要点 → 提取相关表述 → 0-5分评分
    ↓
步骤2: 逻辑评估 (Logic Assessment)
  整体推理连贯性 → 推理深度 → 0-5分评分
    ↓
步骤3: 综合反馈 (Comprehensive Feedback)
  计算总分 → 苏格拉底式引导提问 → 标注薄弱知识点标签
    ↓
结构化 JSON 输出 → Pydantic 校验 → 前端展示
```
Описание каждого шага: ...

#### 3.2.2 少样本提示 (Few-Shot Prompting)
- 在 Prompt 中嵌入示例 JSON 输出格式
- 降低 LLM 输出格式错误的概率
- 配合 PydanticOutputParser 实现双重保障

### 3.3 多角色权限管理 / Управление ролями и правами

#### 3.3.1 认证流程 / Процесс аутентификации
```
登录页 → 选择角色(教师/学生) → 输入用户名密码
  → SHA256(password) 与数据库比对
  → 成功: st.session_state 写入用户信息
  → 失败: 错误提示, 重新输入
```
Описание: ...

#### 3.3.2 角色隔离 / Изоляция ролей
| 功能 / Функция | 教师 / Преподаватель | 学生 / Студент |
|---------------|---------------------|---------------|
| 教材管理 | 上传 PDF, 构建知识库 | 不可见 |
| 习题生成 | RAG 出题, 发布习题 | 不可见 |
| 学生成绩 | 查看全部学生记录 | 仅查看本人 |
| 习题作答 | 不可见 | 答题 + 获取批改 |
| 错题集 | 不可见 | 查看个人薄弱点 |
| 师生聊天 | 选择学生, 1v1 沟通 | 选择教师, 1v1 沟通 |

### 3.4 实时聊天系统 / Система чата в реальном времени

#### 3.4.1 技术实现 / Техническая реализация
- 消息存储：SQLite messages 表
- "实时"机制：`@st.fragment(run_every=2)` 每2秒局部刷新消息区
- 文件传输：Streamlit file_uploader → 保存至 uploads/chat/ → 数据库记录路径
- 消息渲染：自定义 HTML/CSS 微信风格气泡

#### 3.4.2 支持的文件类型 / Поддерживаемые типы файлов

- 文档（pdf/docx/txt/pptx）→ 文件名链接

### 5.1 代码层改进 / Улучшения уровня кода

- 优势：更好的性能、自定义 UI、WebSocket 原生支持 / Преимущества: лучшая производительность, кастомный UI, нативная поддержка WebSocket

#### 5.1.3 代码重构 / Рефакторинг кода

### 5.2 实现层改进 / Улучшения уровня реализации

#### 5.2.1 真正的 WebSocket 实时通信 / Настоящая real-time коммуникация через WebSocket
- 现状：2秒轮询 (st.fragment run_every=2)
- 改进：FastAPI WebSocket 推送
- 优势：零延迟、减少数据库查询、支持在线状态

#### 5.2.2 多模型横向对比 / Сравнительный анализ нескольких моделей
- 接入模型：DeepSeek-V3, GLM-4, GPT-4o, Claude
- 对比维度：出题质量、批改准确率、格式遵循率、响应速度
- 评估方法：RAGAS 框架 (Context Relevance + Faithfulness)

#### 5.2.3 混合检索策略 / Гибридная стратегия поиска
- 现状：纯 Dense Retrieval (向量相似度)
- 改进：Dense + Sparse 混合检索 (BM25 + Embedding)

#### 5.2.4 知识追踪 (Knowledge Tracing) / Отслеживание знаний
- 现状：基于 Tags 的简单错误计数
- 改进：引入 BKT (Bayesian Knowledge Tracing) 或 DKT (Deep Knowledge Tracing)
- 目的：动态评估学生知识掌握概率
